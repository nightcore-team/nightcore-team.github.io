## /config clans setup
!!! info ""
    Команда для общей настройки системы кланов

    ### Доступ
    `CLANS_CONFIG_ACCESS`

    ### Параметры
    - `shop_threads_channel` - Канал, под которым создаются ветки с покупками. (опционален)
    - `shop_buy_ping_roles` - Роли для упоминания при покупке в магазине. Формат: role_id, role_id, ... (опционален)
    - `shop_items` - Товары, доступные в магазине. Формат: item_name, price | item_name, price | ... (опционален)
    - `reputation_per_payday` - Количество репутации, выдаваемой в пейдэй. (опционален)
    - `payday_channel` - Канал для объявлений о дне зарплаты. (опционален)
    - `base_exp_multiplier` - Базовый множитель опыта, выдаваемого за сообщение. (опционален)
    - `improvements_costs` - Стоимость улучшений клана (Всего их 3). Формат: cost,cost,cost (опционален)

## /config levels setup
!!! info ""
    Команда для настройки системы системы уровней

    ### Доступ
    `LOGGING_CONFIG_ACCESS`

    ### Параметры
    - `count_messages_channel` - Канал для подсчета сообщений для уровней, текстовый (опционален)
    - `count_messages_type` - Тип подсчета сообщений для уровней: `all` (Все каналы) или `channel_only` (Только указанный) (опционален)

    - `coins_multiplier` - Множитель монет для уровней. (опционален)
    - `battlepass_multiplier` - Множитель опыта для боевого пропуска. (опционален)
    - `roles_with_bonus` - Роли, которые получают бонусные очки опыта. Формат: role_id, multiplier | ... (опционален)
    - `roles_per_level` - Роли, назначаемые на каждом уровне. Формат: level, role_id | level, role_id | ... (опционален)

## /config moderation setup
!!! info ""

    Настроить систему модерации

    ### Доступ
    `MODERATION_CONFIG_ACCESS`

    ### Параметры
    - `moderation_access_roles` - Роли, которые могут получать доступ к функциям модерации (`MODERATION_ACCESS`), Формат: id роли,id роли,... (опционален)
    - `leadership_access_roles` - Роли, которые могут получать доступ к функциям модерации для руководства. (`HEAD_MODERATION_ACCESS`), Формат: id роли,id роли,... (опционален)
    - `ban_access_roles` - Роли, которые могут получать доступ к бану (`BAN_ACCESS`), Формат: id роли,id роли,... (опционален)
    - `unban_access_roles` - Роли, которые могут получать доступ к разбану (`UNBAN`), Формат: id роли,id роли,...  (опционален)
    - `ban_request_ping_role` - Роль, которую нужно пинговать, когда создается запрос на бан. (Упоминание или id) (опционален)
    - `ban_request_channel` - Канал, в котором создаются запросы на бан, текстовый (опционален)
    - `mute_type` - Тип мута, который нужно применить по команде `/mute`: Timeout | Role (опционален)
    - `mute_role` - Роль, которую нужно назначить, когда пользователь замьючен. (Упоминание или id) (опционален)
    - `mpmute_role` - Роль, которую нужно назначить, когда пользователь замьючен в торговой площадке. (Упоминание или id) (опционален)
    - `vmute_role` - Роль, которую нужно назначить, когда пользователь замьючен в голосовом канале. (Упоминание или id) (опционален)
    - `leaders_access_rr_roles` - Роли лидеров с доступом к команде /rr, Формат: id роли, id роли,... (опционален)
    - `fraction_roles_access` - Роли, с доступом к команде `/fraction_role`, формат: id роли для выдачи, id роли с доступом_id роли с доступом_... | ... (опционален)

## /config battlepass add_level
!!! info ""

    Команда для добавления уровня в батлпасс

    Типы наград:

    - `EXP` - опыт
    - `COINS` - коины
    - `COLOR` - цвет, выбор нужного в параметре `reward`
    - `CASE` - кейс, выбор нужного в параметре `reward`
    - `CUSTOM` - кастомная награда в виде строки, ввод в параметр `reward`
    - `BATTLEPASS_POINTS` - очки батлпасса

    ### Доступ
    `ECONOMY_CONFIG_ACCESS`

    ### Параметры
    - `exp_required` - Количество `EXP` для получения этого уровня
    - `reward_type` - Тип награды
    - `reward_amount` - Количество, игнорируется для типа награды `COLOR`
    - `reward` - Выбор кейса / цвета / ввод текста, в зависимости от типа награды  (опционален)  
    Если тип награды не подходящий, вы увидите один вариант ввода - `Данный параметр используется только для типов кейс/цвет / custom!` (опционален)
    - `before_level` - Номер уровня для добавления нового перед ним, без ввода - уровень добавляется в конец (опционален)
