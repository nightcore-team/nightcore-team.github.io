# Команды системы билдера компонентов

---

## /components create
!!! info ""
    Создать новый компонент, который сохранится в базе данных бота

    После ввода команды вы увидите модальное окно с параметрами:

    - `Название компонента`
    - `Выбор типа компонента` - Embed или V2
    - `Текст компонента`
    - `Авторство`

    ### Доступ
    `MODERATION_ACCESS`

---

## /components change
!!! info ""
    Изменить существующий компонент

    После ввода команды вы увидите модальное окно с параметрами:

    - `Название компонента` (опционален)
    - `Выбор типа компонента` - Embed или V2 (опционален)
    - `Текст компонента` (опционален)
    - `Авторство` (опционален)

    ### Доступ
    `MODERATION_ACCESS`

    ### Параметры
    - `component` - название компонента  
    Вывод всех компонентов для выбора с помощью автокомплита  
    Как значение передается название компонента

!!! warning "Возможные ошибки"
    - Выбранный компонент не найден

---

## /components preview
!!! info ""
    Отправить компонент видимым только себе

    После ввода команды вы увидите модальное окно с параметрами:

    - `Ссылка на изображение` - ограничение - до 300 символов (опционален)
    - `Изображение с устройства` - параметр для загрузки изображения (опционален)


    ### Доступ
    `MODERATION_ACCESS`

    ### Параметры
    - `component` - название компонента  
    Вывод всех компонентов для выбора с помощью автокомплита  
    Как значение передается название компонента
    - `color` - цвет компонента в HEX, пример для красного: FF0000  (опционален)

!!! warning "Возможные ошибки"
    - Выбранный компонент не найден

---

## /components send
!!! info ""
    Отправить компонент видимым только себе  
    Если вы указали роль, то она отправится отдельным сообщением перед сообщением с компонентом

    После ввода команды вы увидите модальное окно с параметрами:

    - `Ссылка на изображение` - ограничение - до 300 символов (опционален)
    - `Изображение с устройства` - параметр для загрузки изображения (опционален)


    ### Доступ
    `MODERATION_ACCESS`

    ### Параметры
    - `component` - название компонента  
    Вывод всех компонентов для выбора с помощью автокомплита  
    Как значение передается название компонента
    - `channel` - канал для отправки компонента, текстовый
    - `role` - роль для упоминания вместе с отправленным компонентом (Упоминание или id) (опционален)
    - `color` - цвет компонента в HEX, пример для красного: FF0000  (опционален)

!!! warning "Возможные ошибки"
    - Выбранный компонент не найден
