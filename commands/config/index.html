    Команды системы конфигурации

    /config clans setup

    Команда для общей настройки системы кланов

    Доступ

    CLANS_CONFIG_ACCESS

    Параметры

    • shop_threads_channel - Канал, под которым создаются ветки с покупками. (опционален)
    • shop_buy_ping_roles - Роли для упоминания при покупке в магазине. Формат: role_id, role_id, ... (опционален)
    • shop_items - Товары, доступные в магазине. Формат: item_name, price | item_name, price | ... (опционален)
    • reputation_per_payday - Количество репутации, выдаваемой в пейдэй. (опционален)
    • payday_channel - Канал для объявлений о дне зарплаты. (опционален)
    • base_exp_multiplier - Базовый множитель опыта, выдаваемого за сообщение. (опционален)
    • clan_category - Категория в которой будет создаваться текстовый канал (опционален)
    • improvements_costs - Стоимость улучшений клана (Всего их 3). Формат: cost,cost,cost (опционален)

    /config clans update_clans_access

    Команда для настройки доступа CLANS_ACCESS

    Параметры

    • role - роль для настройки доступа (Упоминание или id)
    • option - выбор из add (выдать доступ) и remove (снять доступ)

    /config tickets

    Команда для настройки системы тикетов

    Доступ

    MODERATION_CONFIG_ACCESS

    Параметры

    • new_tickets_category - Категория для новых тикетов (Упоминание или id) (опционален)
    • pinned_tickets_category - Категория для закрепленных тикетов (Упоминание или id) (опционален)
    • closed_tickets_category - Категория для закрытых тикетов (Упоминание или id) (опционален)
    • create_ticket_ping_role - Роль для упоминания при создании тикета (Упоминание или id) (опционален)

    /config logging

    Команда для настройки системы логирования

    Доступ

    LOGGING_CONFIG_ACCESS

    Параметры

    • bans - Канал для логирования всех банов в гильдии. (опционален)
    • voices - Канал для логирования изменений голосового состояния (выход из канала, вход, переход в другой канал). (опционален)
    • members - Канал для логирования обновлений участников. (опционален)
    • channels - Канал для логирования обновлений каналов. (опционален)
    • roles - Канал для логирования обновлений ролей. (опционален)
    • messages - Канал для логирования обновлений сообщений. (опционален)
    • moderation - Канал для логирования действий модерации. (опционален)
    • tickets - Канал для логирования обновлений тикетов. (опционален)
    • reactions - Канал для логирования обновлений реакций. (опционален)
    • private_rooms - Канал для логирования обновлений приватных комнат. (опционален)
    • economy - Канал для логирования обновлений экономики. (опционален)
    • clan - "Канал для логирования обновлений кланов. (опционален)
    • ignoring_channels - Каналы для игнорирования при логировании. Формат: id,id,id,... (опционален)

    /config logging update_ignoring_channels

    Команда для настройки каналов, в которых не будут логироваться сообщения

    Доступ

    LOGGING_CONFIG_ACCESS

    Параметры

    • channel - текстовый канал (Упоминание или id)
    • option - выбор из add (добавить в список игнорируемых) и remove (убрать из списка игнорируемых)

    Возможные ошибки

    • Конфигурация системы логирования не создана (Введите /config logging setup)

    /config levels setup

    Команда для настройки системы системы уровней

    Доступ

    LOGGING_CONFIG_ACCESS

    Параметры

    • count_messages_channel - Канал для подсчета сообщений для уровней, текстовый (опционален)
    • count_messages_type - Тип подсчета сообщений для уровней: all (Все каналы) или channel_only (Только указанный) (опционален)
    • level_notify_channel - Канал для уведомлений о повышении уровня, текстовый (Упоминание или id) (опционален)
    • exp_multiplier - Множитель опыта для уровней. (опционален)
    • coins_multiplier - Множитель монет для уровней. (опционален)
    • battlepass_multiplier - Множитель опыта для боевого пропуска. (опционален)
    • roles_with_bonus - Роли, которые получают бонусные очки опыта. Формат: role_id, multiplier | ... (опционален)
    • roles_per_level - Роли, назначаемые на каждом уровне. Формат: level, role_id | level, role_id | ... (опционален)

    /config private_channels

    Настроить канал для создания приватных комнат

    Доступ

    PRIVATE_CHANNELS_CONFIG_ACCESS

    Параметры

    • create_private_channel - голосовой канал (Упоминание или id) (опционален)

    /config moderstats

    Настроить баллы за наказания, роль для подсчета баллов и канал для подсчета сообщений

    Доступ

    MODERATION_CONFIG_ACCESS

    Параметры

    • mute - баллы за выдачу /mute (опционален)
    • ban - баллы за выдачу /ban (опционален)
    • kick - баллы за /kick (опционален)
    • ticket - баллы за закрытие тикета (опционален)
    • vmute - баллы за /vmute (опционален)
    • mpmute - баллы за /mpmute (опционален)
    • ticket_ban - баллы за /ticketban (опционален)
    • role_request - баллы за принятие запроса на получение роли (опционален)
    • role_remove - баллы за /rr (опционален)
    • message - баллы за сообщение (опционален)
    • notification - баллы за /notify (опционален)
    • role - Роль для отслеживания статистики модерации (Упоминание или id) (опционален)
    • channel - Канал для подсчета сообщений модераторов, текстовый (Упоминание или id) (опционален)

    /config notifications

    Настроить систему оповещений

    Доступ

    NOTIFICATIONS_CONFIG_ACCESS

    Параметры

    • notifications - Канал для отправки уведомлений (/notify), текстовый или ветка (Упоминание или id) (опционален)
    • moderation_notifications - Канал для отправки уведомлений для модерации, текстовый или ветка (Упоминание или id) (опционален)
    • nightcore_notifications - канал для отправки уведомлений от Nightcore, текстовый или ветка (Упоминание или id) (опционален)

    /config infomaker setup

    Настроить логирования для инфомейнеров

    Доступ

    INFOMAKER_CONFIG_ACCESS

    Параметры

    • admins_roles - роли администраторов, которые будут логироваться. Формат: id роли,id роли,id роли,... (опционален)
    • leaders_roles - роли лидеров, которые будут логироваться. Формат: id роли,id роли,id роли,... (опционален)
    • admins_logging_channel - канал, в который будут отправляться логи административных ролей, текстовый (Упоминание или id) (опционален)
    • leaders_logging_channel - канал, в который будут отправляться логи лидерских ролей, текстовый (Упоминание или id) (опционален)

    /config other setup

    Настроить остальные системы

    Доступ

    OTHER_CONFIG_ACCESS

    Параметры

    • rules_channel - канал для правил, текстовый (Упоминание или id) (опционален)
    • proposal_channel - Канал для предложений, текстовый (Упоминание или id) (опционален)
    • illegal_roles - роли нелегалов. Формат: название организации, тег, id роли | название организации, тег, id роли | ... (опционален)
    • organizational_roles - Организационные роли. Формат: название организации, тег, id роли | название организации, тег, id роли | ... (опционален)
    • role_request_channel - Канал для проверки запросов на роли, текстовый (Упоминание или id) (опционален)

    /config economy setup

    Настроить систему экономики

    Доступ

    ECONOMY_CONFIG_ACCESS

    Параметры

    • casino_channel - Канал для отправки мультиплеерных игр в казино, текстовый (Упоминание или id) (опционален)
    • shop_buy_ping_roles - Роли для упоминания при покупке в магазине, Формат: id роли, id роли (опционален)
    • economy_access_roles - Роли с доступом к командам экономики (ECONOMY_ACCESS), Формат: id роли, id роли (опционален)
    • shop_items - Конфигурация предметов в магазине. Формат: название, цена | название, цена | ... (опционален)
    • reward_bonus - Количество коинов, выдаваемое в /reward (опционален)
    • coin_name - Название валюты, ограничение - 100 символов (опционален)
    • color_compensation - Компенсация за существующий у юзера цвет, в коинах (опционален)

    /config economy update_economy_access

    Команда для настройки доступа ECONOMY_ACCESS

    Доступ

    ECONOMY_CONFIG_ACCESS

    Параметры

    • role - роль для настройки доступа (Упоминание или id)
    • option - выбор из add (выдать доступ) и remove (снять доступ)

    Возможные ошибки

    • Конфигурация системы экономики не создана (Введите /config economy setup)

    /config moderation setup

    Настроить систему модерации

    Доступ

    MODERATION_CONFIG_ACCESS

    Параметры

    • moderation_access_roles - Роли, которые могут получать доступ к функциям модерации (MODERATION_ACCESS), Формат: id роли,id роли,... (опционален)
    • leadership_access_roles - Роли, которые могут получать доступ к функциям модерации для руководства. (HEAD_MODERATION_ACCESS), Формат: id роли,id роли,... (опционален)
    • ban_access_roles - Роли, которые могут получать доступ к бану (BAN_ACCESS), Формат: id роли,id роли,... (опционален)
    • unban_access_roles - Роли, которые могут получать доступ к разбану (UNBAN), Формат: id роли,id роли,... (опционален)
    • ban_request_ping_role - Роль, которую нужно пинговать, когда создается запрос на бан. (Упоминание или id) (опционален)
    • ban_request_channel - Канал, в котором создаются запросы на бан, текстовый (опционален)
    • mute_type - Тип мута, который нужно применить по команде /mute: Timeout | Role (опционален)
    • mute_role - Роль, которую нужно назначить, когда пользователь замьючен. (Упоминание или id) (опционален)
    • mpmute_role - Роль, которую нужно назначить, когда пользователь замьючен в торговой площадке. (Упоминание или id) (опционален)
    • vmute_role - Роль, которую нужно назначить, когда пользователь замьючен в голосовом канале. (Упоминание или id) (опционален)
    • leaders_access_rr_roles - Роли лидеров с доступом к команде /rr, Формат: id роли, id роли,... (опционален)
    • fraction_roles_access - Роли, с доступом к команде /fraction_role, формат: id роли для выдачи, id роли с доступом_id роли с доступом_... | ... (опционален)

    /config moderation update_access

    Команда для настройки доступов к модерации:

    • Руководство модерации (HEAD_MODERATION_ACCESS)
    • Модерация (MODERATION_ACCESS)
    • Команда /ban (BAN_ACCESS)
    • Команда /unban (UNBAN_ACCESS)
    • Команда /rr

    Доступ

    MODERATION_CONFIG_ACCESS

    Параметры

    • type - тип доступа для настройки
    • role - роль для настройки доступа (Упоминание или id)
    • option - выбор из add (выдать доступ) и remove (снять доступ)

    /config moderation update_fraction_role_access

    Команда для настройки доступа к /fraction_role

    Доступ

    MODERATION_CONFIG_ACCESS

    Параметры

    • access_role - роль которой меняется доступ (Упоминание или id)
    • fraction_role - роль для которой меняется доступ (Упоминание или id)
    • option - выбор из add (выдать доступ) и remove (снять доступ)

    /config battlepass reset

    Команда для сброса у всех пользователей очков батлпасса

    Доступ

    ECONOMY_CONFIG_ACCESS

    /config battlepass add_level

    Команда для добавления уровня в батлпасс

    Типы наград:

    • EXP - опыт
    • COINS - коины
    • COLOR - цвет, выбор нужного в параметре reward
    • CASE - кейс, выбор нужного в параметре reward
    • CUSTOM - кастомная награда в виде строки, ввод в параметр reward
    • BATTLEPASS_POINTS - очки батлпасса

    Доступ

    ECONOMY_CONFIG_ACCESS

    Параметры

    • exp_required - Количество EXP для получения этого уровня
    • reward_type - Тип награды
    • reward_amount - Количество, игнорируется для типа награды COLOR
    • reward - Выбор кейса / цвета / ввод текста, в зависимости от типа награды (опционален) Если тип награды не подходящий, вы увидите один вариант ввода - Данный параметр используется только для типов кейс/цвет / custom! (опционален)
    • before_level - Номер уровня для добавления нового перед ним, без ввода - уровень добавляется в конец (опционален)

    /config battlepass delete_level

    Команда для удаления уровня из батлпасса

    Доступ

    ECONOMY_CONFIG_ACCESS

    Параметры

    level - номер уровня для удаления, ограничение - от 1 до 1000000

    Возможные ошибки

    • Уровень в данным номером не найден

    /config battlepass change_level

    Команда для изменения существующего уровня батлпасса

    Типы наград:

    • EXP - опыт
    • COINS - коины
    • COLOR - цвет, выбор нужного в параметре reward
    • CASE - кейс, выбор нужного в параметре reward
    • CUSTOM - кастомная награда в виде строки, ввод в параметр reward
    • BATTLEPASS_POINTS - очки батлпасса

    Доступ

    ECONOMY_CONFIG_ACCESS

    Параметры

    • level - номер уровня для удаления, ограничение - от 1 до 1000000
    • new_required_exp - Количество EXP для получения этого уровня (опционален)
    • new_reward_type - Тип награды (опционален)
    • new_reward_amount - Количество, игнорируется для типа награды COLOR (опционален)
    • new_reward - Выбор кейса / цвета / ввод текста, в зависимости от типа награды (опционален) Если тип награды не подходящий, вы увидите один вариант ввода - Данный параметр используется только для типов кейс/цвет / custom! (опционален)

    Возможные ошибки

    • Уровень в данным номером не найден
    • Вы не выбрали ни одного параметра для изменения.

    /system config info

    Команда для просмотра конфигурации определенной системы:

    • Логирование - система логирования
    • Экономика - система экономики
    • Уровни - система уровней
    • Кланы - система кланов
    • Приватные каналы - система приватных каналов
    • Модерация - система модерации
    • Уведомления - система оповещений бота
    • Инфомейкеры - система оповещений инфомейкеров
    • Другое - прочие системы

    Доступ

    ADMINISTRATOR

    Параметры

    • system - Название системы, информацию о которой вы хотите получить
    • ephemeral - видимость ответа бота True (Видно всем) или False (Видно только тому кто ввел команду) (опционален)

    /system config setup_access

    Команда для первичной настройки доступов к системам конфигурации:

    Доступ

    ADMINISTRATOR

    Параметры

    • other - Список ролей с доступом к основной конфигурации. Формат: role_id,role_id,... (опционален)
    • clans - Список ролей с доступом к системе кланов. Формат: role_id,role_id,... (опционален)
    • economy - Список ролей с доступом к системе экономики. Формат: role_id,role_id,... (опционален)
    • levels - Список ролей с доступом к системе уровней. Формат: role_id,role_id,... (опционален)
    • logging - Список ролей с доступом к системе логов. Формат: role_id,role_id,... (опционален)
    • moderation - Список ролей с доступом к системе модерации. Формат: role_id,role_id,... (опционален)
    • notifications - Список ролей с доступом к системе уведомлений. Формат: role_id,role_id,... (опционален)
    • private_channels - Список ролей с доступом к системе приватных каналов. Формат: role_id,role_id,... (опционален)
    • infomaker - Список ролей с доступом к системе инфомейкера. Формат: role_id,role_id,... (опционален)

    /system config update_access

    Команда для настройки доступов к определенной системе:

    • Все - Доступ выдается для всех систем
    • Логирование - система логирования (LOGGING_CONFIG_ACCESS)
    • Экономика - система экономики (ECONOMY_CONFIG_ACCESS)
    • Уровни - система уровней (LEVELS_CONFIG_ACCESS)
    • Кланы - система кланов (CLANS_CONFIG_ACCESS)
    • Приватные каналы - система приватных каналов (PRIVATE_CHANNELS_CONFIG_ACCESS)
    • Модерация - система модерации (MODERATION_CONFIG_ACCESS)
    • Уведомления - система оповещений бота (NOTIFICATIONS_CONFIG_ACCESS)
    • Инфомейкеры - система оповещений инфомейкеров (INFOMAKER_CONFIG_ACCESS)
    • Другое - прочие системы (OTHER_CONFIG_ACCESS)

    Доступ

    ADMINISTRATOR

    Параметры

    • system- Система для обновления доступа
    • role - роль для которой меняется доступ (Упоминание или id)
    • option - выбор из add (выдать доступ) и remove (снять доступ)
